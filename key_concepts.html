What is HTTP?

The Hypertext Transfer Protocol (HTTP) is the application-level protocol that powers the World Wide Web. It defines how a client (such as a browser) requests resources and how a server responds with the requested data. Every time you click a link or type a URL, your device uses HTTP to retrieve text, images, video, and more.

Request/Response Model

HTTP communication follows a simple request/response pattern:

  • Request: The client sends a message specifying the action to perform (for example, GET or POST) and the target URL.
  • Response: The server replies with a status code, headers, and an optional body that contains the requested content or an error message.

Status codes such as 200 OK or 404 Not Found tell the client whether the request was successful. Headers provide metadata such as content type and caching rules.

Common Methods

HTTP defines many methods to describe the desired action. Some of the most common are:

  • GET: Retrieve a resource without modifying it. The most frequently used method.
  • POST: Submit data to the server, often resulting in a change of state or a side effect.
  • PUT: Replace a resource at a given URL with the data provided by the client.
  • DELETE: Remove the specified resource from the server.

Important Concepts

  • URL (Uniform Resource Locator): The address used to locate a resource on the web.
  • Headers: Key value pairs describing the request or response, caching rules, and more.
  • Statelessness: Each HTTP request is independent; servers do not retain session state between requests unless additional mechanisms like cookies are used.

By mastering these basics, it becomes easier to understand the evolution of HTTP, the security improvements brought by HTTPS, and the modern performance enhancements covered on the other pages of this site.
